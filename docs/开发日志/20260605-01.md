## 功能开发

### 1. 悬浮窗拖动与位置记忆

**需求：** 悬浮窗支持拖动位置，并记住位置重启后恢复

**实现：**
- 使用 Tauri 的 `data-tauri-drag-region` 属性实现窗口拖动
- 监听 `onMoved` 事件，实时保存位置到数据库
- 启动时从数据库恢复上次位置

**修改文件：**
- `src/types/settings.ts` - 添加 `floatingX`、`floatingY` 字段
- `src/stores/settingsStore.ts` - 处理新字段的加载/保存
- `src/composables/useSettings.ts` - 同步更新
- `src/components/floating/FloatingWindow.vue` - 实现拖动和位置记忆

### 2. 悬浮窗固定/解锁功能

**需求：** 支持固定悬浮窗位置，固定后隐藏边框，可通过托盘解锁

**实现：**
- 未固定状态：显示边框 + 半透明背景，可拖动

- 左上角图钉按钮（未固定时斜的，固定时竖的）
- 托盘右键菜单"固定悬浮窗"，已固定时显示勾选

**修改文件：**
- `src/types/settings.ts` - 添加 `floatingPinned` 字段
- `src/stores/settingsStore.ts` - 处理新字段
- `src/components/floating/FloatingWindow.vue` - 实现固定/解锁功能
- `src-tauri/src/utils/tray.rs` - 使用 CheckMenuItem，监听状态变化

## Bug 修复

### 1. 跨窗口动画失效

**问题：** 新建 TODO 后悬浮窗没有动画效果

**原因：** 跨窗口 Pinia Store 隔离，`newTodoId` 只在管理窗口的 store 中设置

**解决：**
- Rust 后端：`todo-changed` 事件携带新 TODO 的 ID
- FloatingWindow：监听事件时设置 `newTodoId` 到悬浮窗的 store

**修改文件：**
- `src-tauri/src/commands/todo_commands.rs` - emit 携带 ID
- `src/components/floating/FloatingWindow.vue` - 接收 ID 并设置

---

### 2. 托盘固定状态不同步

**问题：** 固定悬浮窗后托盘菜单没有显示勾选

**原因：** 发送的是字符串 "true"/"false"，但 Tauri 事件序列化会加引号

**解决：** 直接发送布尔值，Rust 端用 `event.payload() == "true"` 解析

**修改文件：**
- `src/components/floating/FloatingWindow.vue` - 发送布尔值
- `src-tauri/src/utils/tray.rs` - 修改解析方式

## 技术笔记

### Tauri v2 窗口 API

- `data-tauri-drag-region` - 实现窗口拖动
- `getCurrentWindow().onMoved()` - 监听窗口移动
- `getCurrentWindow().setPosition()` - 设置窗口位置
- `PhysicalPosition` - 物理像素位置

### 跨窗口事件通信

- `emitTo('window-name', 'event', payload)` - 发送到特定窗口
- `emit('event', payload)` - 发送到 app 级别（托盘可接收）
- `listen('event', callback)` - 监听事件

### CheckMenuItem

```rust
use tauri::menu::CheckMenuItem;

let item = CheckMenuItem::with_id(app, "id", "text", enabled, checked, accelerator)?;
item.set_checked(true)?; // 更新勾选状态
```
