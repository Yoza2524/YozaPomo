## 悬浮窗 TODO 动画优化

### 问题
1. 新建第 2、3 个 TODO 时悬浮窗闪烁
2. 创建超出上限的 TODO 时，省略号区域需要缓慢出现/消失

### 闪烁根因分析
多个问题叠加：
1. **`fetchTodayTodos` 设置 `loading=true`** — 导致 TodoList 组件从 `v-if="loading"` 切换到 `v-else`，整个列表销毁再重建
2. **ResizeObserver 在 `animationstart` 之前触发** — 此时 `isAnimating` 还是 false，debounce 回调读到动画前的瞬间旧数据（最小高度），窗口被砍到 124px 再涨回来
3. **`resizeImmediate` 串行异步** — `outerSize().then(outerPosition().then(...))` 延迟叠加

### 修复内容
1. **`todoStore.ts`** — `fetchTodayTodos` 已有数据时不设 `loading`，避免组件销毁重建
2. **`useAutoResize.ts`** — 重构为动画计数器 + rAF 驱动：
   - 动画期间由 `animationstart`/`animationend` 控制 `animCount`
   - ResizeObserver 回调和 debounce 回调都检查 `animCount`，动画期间完全跳过
   - 动画期间由 rAF 循环驱动 resize，动画结束后由 ResizeObserver 接管
   - `resizeImmediate` 改用 `Promise.all` 并行获取尺寸和位置
   - 改用 `getCurrentWindow()` 替代 `getCurrentWebviewWindow()`
3. **`TodoTag.vue`** — 列表已满时新建用 `todoFadeIn`（transform 动画，不占高度），未满时用 `todoExpand`（height 展开）
4. **`TodoList.vue`** — 省略号首次出现用 `ellipsisIn` 动画，消失用 `ellipsisOut` 动画，中间变化无动画
5. **`FloatingWindow.vue`** — `listen<string | null>` 兼容 Rust 端 `()` payload（删除/更新事件）
6. **`OverflowAnim.vue`** — `position: absolute` 脱离文档流，避免撑大父容器高度
7. **`TodoList.vue`** — 省略号出现用 transform 滑入（不占高度），消失用 height 动画收起（缓慢缩回高度）
8. **`TodoTag.vue`** — 增强呼吸灯效果：边框颜色白→绿渐变，box-shadow 扩散，background-clip 防遮挡
9. **`FloatingWindow.vue`** — 底部操作区 padding 对齐 TODO 列表（px-3）
10. **`FocusButton.vue`** — 去掉 gap，margin 移到倒计时块显示时；统一圆角为 12px
11. **`TodoContextMenu.vue`** — 增加"结束专注"菜单项；"结束TODO"改为"完成待办"（绿色对勾）；菜单自动检测右边界防裁剪

## 删除动画：粉碎粒子效果

### 需求
删除 TODO 时 TodoTag 出现粉碎效果，变成粒子自然坠落，然后下方内容缓慢上升。

### 实现方案
纯色方块 + requestAnimationFrame 物理模拟，不引入外部依赖。

### 实现流程
1. 用户在管理界面删除 TODO → Rust `delete_todo` emit `todo-deleted` 事件（携带 ID）
2. 悬浮窗监听 `todo-deleted` → 设置 `shatterTodoId`
3. TodoTag 监听 `shatter` prop → Tag 内容淡出（0.15s）+ 高度从上往下收起（0.4s）+ 18 个粒子飞散坠落（0.4s）
4. 粒子动画结束 → `shatterDone` 事件 → FloatingWindow 刷新列表

### 修改文件
1. **`todo_commands.rs`** — `delete_todo` 改为 emit `todo-deleted` 事件（携带 ID），替代原 `todo-changed`
2. **`TodoTag.vue`** — 新增 `shatter` prop、`shatterDone` 事件、`startShatter()` 粒子物理动画（重力/随机方向/旋转/渐隐）
3. **`TodoList.vue`** — 透传 `shatterTodoId` prop 和 `shatterDone` 事件
4. **`FloatingWindow.vue`** — 监听 `todo-deleted` 事件触发粉碎动画，`handleShatterDone` 刷新列表；"完成待办"流程保持不变

## 删除动画重写：三种场景支持

### 需求
1. **无省略号删除**：粒子特效 → tag 收起（height 40→0）→ 下方内容上移 → 窗口缩小（同步）
2. **有省略号删除**：粒子特效 → tag 收起 → 从省略号向上展开新 tag 填充位置 → 总高度不变
3. **最后一个 tag 删除**：粒子特效 → tag 收起 → "暂无待办"同步出现（总高度不变）

### 实现方案

**核心思路**：用 CSS transition 驱动所有高度变化，`useAutoResize` 的 ResizeObserver 自然捕获，窗口跟随调整。

**动画参数**：统一 400ms，`cubic-bezier(0.4, 0, 0.2, 1)`

### 修改文件

1. **`TodoTag.vue`**
   - 新增 `collapsing` prop：触发收起动画（height 40→0，opacity 1→0）
   - 新增 `appearing` prop：触发展开动画（height 0→40）
   - 新增 `collapseDone` / `appearDone` 事件
   - 粒子动画完成后不再 emit，收起由 CSS transition 驱动
   - `onTransitionEnd` 监听 height 变化触发 `collapseDone`
   - `onAnimationEnd` 监听 appearing 动画完成触发 `appearDone`

2. **`TodoList.vue`**
   - 管理动画状态机：`collapsingTodoId`、`appearingTodoId`、`ellipsisCollapsing`、`isLastTagCollapsing`
   - 监听 `shatterTodoId` 变化启动动画
   - `handleCollapseDone`：无隐藏项时直接完成，有隐藏项时等待 `appearDone`
   - `handleAppearDone`：收起和展开都完成，通知父组件刷新
   - 省略号收起动画同步（`ellipsis-collapsing` class）
   - "暂无待办"容器固定高度 = 一个 Tag（40px），`fade-sync` 动画与 tag 收起同步

3. **`FloatingWindow.vue`**
   - `handleShatterDone` 接收 `todoId` 参数
   - 刷新列表并清除 `shatterTodoId`

### 数据流
```
管理界面删除 todo
    ↓
Rust emit 'todo-deleted' (携带 todo id)
    ↓
FloatingWindow 设置 shatterTodoId
    ↓
TodoList 计算：
  - collapsingTodoId = shatterTodoId
  - appearingTodoId = 隐藏区域的第一个（如有）
  - isLastTagCollapsing = todos.length <= 1
    ↓
TodoTag 收到 collapsing → 播放粒子 + height 40→0
TodoTag 收到 appearing → height 0→40（从省略号位置）
    ↓
动画结束 → FloatingWindow 刷新列表
```

3. `handleCollapseDone` 开头增加防重复调用检查（`if (!collapsingTodoId.value) return`）
4. `resetAnimState` 中清除兜底定时器

## 计算式窗口高度

### 问题

### 方案

## 修复：useAutoResize 物理/逻辑坐标混用导致窗口飞出屏幕

### 问题
打包到笔记本（1.5x 缩放）后，悬浮窗闪一下就消失了。鼠标移入时窗口也不可见。

### 原因
`resizeImmediate` 中 `outerPosition()` 和 `outerSize()` 返回**物理坐标**，但代码直接拿去减逻辑像素的 `contentWidth`，再传给 `setPosition(LogicalPosition)`。在 1.5x 缩放下，每帧都把窗口往右推 50%，几帧后飞出屏幕。

日志证据：`pos=(1476,60), newPos=(1616,60)` — 屏幕逻辑宽度 1280，窗口被移到 1616。

### 修复
1. **`useAutoResize.ts`** — `resizeImmediate` 和 `updateWindowSize` 中通过 `devicePixelRatio` 将物理坐标转换为逻辑坐标后再计算
2. 容器高度为 0 时跳过 resize（防护首次渲染未完成）
3. 初始 resize 改为 `requestAnimationFrame` 确保首次渲染完成
4. **`lib.rs`** — `set_position` 和 `show` 错误改为 `log::error!` 输出

**状态**：✅ 已提交 2cd618b
