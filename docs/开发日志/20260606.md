# 2026-06-06 开发日志

## 悬浮窗 TODO 动画优化

### 问题
1. 新建第 2、3 个 TODO 时悬浮窗闪烁
2. 创建超出上限的 TODO 时，省略号区域需要缓慢出现/消失

### 闪烁根因分析
多个问题叠加：
1. **`fetchTodayTodos` 设置 `loading=true`** — 导致 TodoList 组件从 `v-if="loading"` 切换到 `v-else`，整个列表销毁再重建
2. **ResizeObserver 在 `animationstart` 之前触发** — 此时 `isAnimating` 还是 false，debounce 回调读到动画前的瞬间旧数据（最小高度），窗口被砍到 124px 再涨回来
3. **`resizeImmediate` 串行异步** — `outerSize().then(outerPosition().then(...))` 延迟叠加

### 修复内容
1. **`todoStore.ts`** — `fetchTodayTodos` 已有数据时不设 `loading`，避免组件销毁重建
2. **`useAutoResize.ts`** — 重构为动画计数器 + rAF 驱动：
   - 动画期间由 `animationstart`/`animationend` 控制 `animCount`
   - ResizeObserver 回调和 debounce 回调都检查 `animCount`，动画期间完全跳过
   - 动画期间由 rAF 循环驱动 resize，动画结束后由 ResizeObserver 接管
   - `resizeImmediate` 改用 `Promise.all` 并行获取尺寸和位置
   - 改用 `getCurrentWindow()` 替代 `getCurrentWebviewWindow()`
3. **`TodoTag.vue`** — 列表已满时新建用 `todoFadeIn`（transform 动画，不占高度），未满时用 `todoExpand`（height 展开）
4. **`TodoList.vue`** — 省略号首次出现用 `ellipsisIn` 动画，消失用 `ellipsisOut` 动画，中间变化无动画
5. **`FloatingWindow.vue`** — `listen<string | null>` 兼容 Rust 端 `()` payload（删除/更新事件）
6. **`OverflowAnim.vue`** — `position: absolute` 脱离文档流，避免撑大父容器高度
7. **`TodoList.vue`** — 省略号出现用 transform 滑入（不占高度），消失用 height 动画收起（缓慢缩回高度）
