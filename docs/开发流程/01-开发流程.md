# 开发流程规范

> 本文档定义 YozaPomo 项目的开发流程、任务管理和代码提交标准。

---

## 1. 开发流程概览

```
需求确认 → 技术方案 → 任务拆分 → 编码实现 → 代码审查 → 测试验证 → 合并发布
    │          │          │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼          ▼          ▼
  PRD文档   技术规范    开发日志    Git提交    Claude审查   本地测试   main分支
```

## 2. 开发阶段

### 阶段零：项目初始化

**目标**：搭建 Tauri + Vue 3 项目骨架

**执行步骤**：
1. 使用 `create-tauri-app` 初始化项目
2. 安装依赖：Naive UI、Tailwind CSS、Pinia、Howler.js
3. 配置 TypeScript 严格模式
4. 配置 ESLint + Prettier
5. 创建基础目录结构
6. 配置 Git `.gitignore`
7. 验证项目可正常启动

**验证标准**：
- [x] `pnpm tauri dev` 能正常启动
- [x] 悬浮窗能显示白色空白窗口
- [x] TypeScript 编译无错误
- [x] ESLint 检查无警告

> 注： 
>  在命令行中Ctrl-c打断时报错

### 阶段一：数据库层

**目标**：实现 SQLite 数据库的完整 CRUD

**执行步骤**：
1. 创建数据库迁移文件（todos、pomodoro_sessions、settings 三张表）
2. 实现 Rust 端数据库初始化
3. 实现 TODO 相关 Tauri 命令（CRUD）
4. 实现专注会话相关 Tauri 命令
   - 创建/查询/更新专注会话
   - 通过 `todo_id` 外键关联 TODO，用于记录每个 TODO 的多次专注时间
   - 支持按 TODO 查询关联的专注历史
5. 实现设置相关 Tauri 命令
6. 前端封装数据库访问 composable

**验证标准**：
- [x] 数据库三张表创建成功
- [x] 能通过 Tauri 命令创建/查询/更新/删除 TODO
- [x] 能通过 Tauri 命令创建/查询专注会话
- [x] 能通过 `todo_id` 查询某个 TODO 关联的所有专注会话
- [x] 设置能正确读写

### 阶段二：悬浮窗基础

**目标**：实现悬浮窗的显示和基本交互

**执行步骤**：
1. 配置悬浮窗窗口属性（透明、无边框、置顶）
2. 实现窗口位置计算（屏幕右上角）
3. 实现悬浮窗主容器组件
4. 实现 TODO 标签组件
   - 显示当天 TODO（数量受设置限制，默认最多 3 个）
   - 超过上限时下方显示省略号
5. 实现开始专注按钮
6. 实现系统托盘
   - 右键菜单：打开管理界面、退出
   - 左键单击：显示/隐藏管理界面

**验证标准**：
- [x] 悬浮窗显示在屏幕右上角
- [x] 悬浮窗背景透明
- [x] 能显示 TODO 标签列表
- [x] 超过设置上限时显示省略号
- [x] 系统托盘图标正常显示
- [x] 托盘右键菜单可用
- [x] 托盘左键单击可显示/隐藏管理界面

### 阶段三：计时器核心

**目标**：实现完整的专注计时逻辑

**执行步骤**：
1. 实现 useTimer composable
2. 实现专注状态管理（Pinia Store）
3. 实现倒计时显示组件
4. 实现进度条组件
5. 实现倒计时归零后的状态转换
   - 倒计时结束 ≠ 专注结束
   - 按钮从"结束专注"变为"开始休息"，并显示本次总专注时长
   - 倒计时切换为超时时长，显示格式：↓xx:xx:xx↓（原定时长）/ ↑xx:xx:xx↑（超时时长）
6. 实现 5 分钟提醒逻辑
   - 专注超过 5 分钟触发，在悬浮窗内播放更小动画 + 更轻提示音
   - 动画消失位置出现"我还在"按钮，1 分钟后自动消失
   - 超时前未点击则强制结束专注，不弹窗，记录为异常专注（status = 'abnormal'）
   - 按钮出现时点击"结束专注"也可结束，"我还在"按钮同时消失
7. 实现超时处理逻辑
8. 实现休息计时逻辑

**验证标准**：
- [ ] 点击"开始专注"后倒计时开始
- [ ] 暂停/恢复功能正常
- [ ] 倒计时归零后按钮变为"开始休息"，显示总专注时长
- [ ] 超时后倒计时显示超时时间（↑xx:xx:xx↑）
- [ ] 专注超过 5 分钟触发提醒动画和提示音
- [ ] "我还在"按钮 1 分钟后自动消失
- [ ] 未点击"我还在"时专注被强制结束，status 记录为 abnormal
- [ ] 点击"结束专注"可结束专注，"我还在"按钮同时消失

### 阶段四：TODO 交互

**目标**：实现 TODO 标签的完整交互

**执行步骤**：
1. 实现 TODO 双击激活/暂停
   - 双击未激活 TODO：开始任务并自动进入番茄钟专注状态
   - 双击已激活 TODO：暂停任务，记录时间片
   - 已有进行中 TODO 时，禁止双击或右键开始其他 TODO
2. 实现 TODO 右键菜单
   - 根据当前状态显示"开始/暂停"
   - 结束：结束任务，Tag 从悬浮窗移除
3. 实现 TODO 与专注绑定
4. 实现 TODO 完成流程（笔记弹窗）
   - 结束 TODO 时弹出 TODO 备注窗口（Markdown 编辑）
   - 结束专注时弹出专注备注窗口（Markdown 编辑）
   - 两者互斥：结束 TODO 只弹 TODO 备注，结束专注只弹专注备注
5. 实现呼吸灯动画效果

**验证标准**：
- [ ] 双击 TODO 能激活并开始专注
- [ ] 双击激活的 TODO 能暂停
- [ ] 已有进行中 TODO 时，无法双击或右键开始其他 TODO
- [ ] 右键菜单功能正确（开始/暂停/结束）
- [ ] 结束 TODO 后 Tag 从悬浮窗移除
- [ ] 结束 TODO 弹出 TODO 备注窗口（Markdown）
- [ ] 结束专注弹出专注备注窗口（Markdown）
- [ ] 呼吸灯动画流畅

### 阶段五：动画与音效

**目标**：实现粒子动画和音效系统

**执行步骤**：
1. 实现 Canvas 粒子动画组件
2. 实现专注结束动画
3. 实现 5 分钟提醒动画
4. 集成 Howler.js 音频播放
5. 实现提示音触发逻辑

**验证标准**：
- [ ] 专注结束时粒子动画播放
- [ ] 5 分钟提醒动画播放
- [ ] 提示音正确播放
- [ ] 动画不影响性能

### 阶段六：管理界面

**目标**：实现完整的管理界面

**执行步骤**：
1. 实现管理界面布局（Naive UI）和页面路由
2. 实现 TODO 管理
   - **今日 TODO** 页面
   - **昨日未完成 TODO** 页面
   - **TODO 列表** 页面（全部历史）
   - TODO 操作：添加（日期、专注时间、备注）、编辑、查看备注、删除
3. 实现专注管理
   - 专注列表页面
   - 编辑操作：开始时间、结束时间、关联 TODO、备注
4. 实现设置页面
   - 打开方式：管理界面顶部 Tab 或托盘右键菜单"设置"
   - 设置项：默认专注时长、显示 TODO 数量、是否显示倒计时数字、提示音频文件
   - 设置修改后立即生效
5. 实现 Markdown 编辑器和渲染器
   - TODO 备注：最上方为 TODO 备注，下方为关联的专注备注
   - 专注备注：独立的 Markdown 编辑

**验证标准**：
- [ ] 管理界面能正常打开
- [ ] 今日 TODO 页面正确显示
- [ ] 昨日未完成 TODO 页面正确显示
- [ ] TODO 列表页面正确显示
- [ ] TODO 增删改查功能完整
- [ ] 专注列表能正确显示
- [ ] 专注记录能编辑（时间、关联 TODO、备注）
- [ ] 设置修改后立即生效
- [ ] Markdown 笔记能正确编辑和渲染
- [ ] TODO 备注页面显示 TODO 备注 + 关联专注备注

### 阶段七：收尾与发布

## 3. 每日开发流程

```
1. 同步代码
   git pull origin main

2. 创建功能分支
   git checkout -b feat/功能名

3. 开发
   - 读取当日开发日志模板
   - 按任务拆分执行
   - 每完成一个小功能就提交

4. 自测
   - 运行 pnpm tauri dev
   - 手动测试相关功能
   - 检查 TypeScript 编译

5. 提交代码
   - 写清楚的 commit message
   - 关联开发日志

6. 合并到 main
   git checkout main
   git merge feat/功能名
   git push origin main
```

```markdown

- [ ] 目标 2

## 完成内容

- ...

## 明日计划

- ...
```

## 5. 代码审查标准

使用 Claude Code 进行代码审查，检查以下维度：

### 5.1 正确性
- [ ] 功能实现是否符合需求文档
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善

### 5.2 代码质量
- [ ] 是否遵循代码规范
- [ ] 命名是否清晰
- [ ] 是否有重复代码
- [ ] 是否有未使用的导入

### 5.3 性能
- [ ] 是否有不必要的重渲染
- [ ] 数据库查询是否高效
- [ ] 内存使用是否合理

### 5.4 安全性
- [ ] SQL 注入防护
- [ ] 用户输入验证
- [ ] 敏感数据保护

## 6. 提交规范

### Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | feat(floating): 添加 TODO 标签组件 |
| fix | 修复 Bug | fix(timer): 修复倒计时不准的问题 |
| docs | 文档更新 | docs: 更新数据库设计文档 |
| style | 代码格式 | style: 格式化 Rust 代码 |
| refactor | 重构 | refactor(store): 重构专注状态管理 |
| test | 测试 | test: 添加 useTimer 单元测试 |
| chore | 构建/工具 | chore: 更新依赖版本 |

### 示例

```
feat(floating): 实现 TODO 标签双击交互

- 实现双击激活 TODO 并开始专注
- 实现双击暂停专注
- 添加呼吸灯动画效果

Closes #12
```
