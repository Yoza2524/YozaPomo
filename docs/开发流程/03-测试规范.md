# 测试规范

> 本文档定义 YozaPomo 项目的测试策略、测试用例和验证标准。

---

## 1. 测试策略

### 1.1 测试金字塔

```
        ┌─────────┐
        │  E2E   │  ← 少量端到端测试（关键流程）
       ┌┴────────┴┐
       │集成测试   │  ← 适量集成测试（组件交互）
      ┌┴──────────┴┐
      │  单元测试   │  ← 大量单元测试（核心逻辑）
      └────────────┘
```

### 1.2 测试工具

| 层级 | 工具 | 用途 |
|------|------|------|
| 单元测试 | Vitest | Vue 组合式函数、工具函数 |
| 组件测试 | Vue Test Utils + Vitest | Vue 组件渲染和交互 |
| E2E 测试 | 手动测试 | 完整用户流程验证 |
| Rust 测试 | `cargo test` | Rust 命令和工具函数 |

### 2.2 测试用例示例

```typescript
// composables/__tests__/useTimer.test.ts
import { describe, it, expect, vi, beforeEach } from 'vitest'
import { useTimer } from '../useTimer'

describe('useTimer', () => {
  beforeEach(() => {
    vi.useFakeTimers()
  })

  it('应该正确初始化计时器', () => {
    const { remainingSeconds, isRunning } = useTimer(25 * 60)

    expect(remainingSeconds.value).toBe(25 * 60)
    expect(isRunning.value).toBe(false)
  })

  it('应该正确开始计时', () => {
    const { remainingSeconds, start, isRunning } = useTimer(25 * 60)

    start()
    expect(isRunning.value).toBe(true)

    vi.advanceTimersByTime(1000)
    expect(remainingSeconds.value).toBe(25 * 60 - 1)
  })

  it('应该正确暂停和恢复', () => {
    const { remainingSeconds, start, pause, resume, isRunning } = useTimer(25 * 60)

    start()
    vi.advanceTimersByTime(5000)
    expect(remainingSeconds.value).toBe(25 * 60 - 5)

    pause()
    expect(isRunning.value).toBe(false)

    vi.advanceTimersByTime(3000)
    expect(remainingSeconds.value).toBe(25 * 60 - 5) // 暂停后不应减少

    resume()
    expect(isRunning.value).toBe(true)

    vi.advanceTimersByTime(2000)
    expect(remainingSeconds.value).toBe(25 * 60 - 7)
  })

  it('应该在倒计时归零时触发回调', () => {
    const onComplete = vi.fn()
    const { start } = useTimer(5, onComplete)

    start()
    vi.advanceTimersByTime(5000)

    expect(onComplete).toHaveBeenCalledTimes(1)
  })
})
```

### 2.3 工具函数测试

```typescript
// utils/__tests__/format.test.ts
import { describe, it, expect } from 'vitest'
import { formatTime, formatDuration } from '../format'

describe('formatTime', () => {
  it('应该正确格式化秒数为 MM:SS', () => {
    expect(formatTime(0)).toBe('00:00')
    expect(formatTime(30)).toBe('00:30')
    expect(formatTime(60)).toBe('01:00')
    expect(formatTime(1500)).toBe('25:00')
    expect(formatTime(3661)).toBe('61:01')
  })

  it('应该处理负数', () => {
    expect(formatTime(-30)).toBe('00:00')
  })
})

describe('formatDuration', () => {
  it('应该正确格式化时长', () => {
    expect(formatDuration(0)).toBe('0 分钟')
    expect(formatDuration(60)).toBe('1 分钟')
    expect(formatDuration(3600)).toBe('1 小时')
    expect(formatDuration(5400)).toBe('1 小时 30 分钟')
  })
})
```

## 4. 集成测试

### 4.1 组件测试示例

```typescript
// components/__tests__/TodoTag.test.ts
import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import TodoTag from '../floating/TodoTag.vue'

describe('TodoTag', () => {
  const mockTodo = {
    id: '1',
    title: '写周报',
    date: '2026-06-04',
    completed: false,
  }

  it('应该正确渲染 TODO 标题', () => {
    const wrapper = mount(TodoTag, {
      props: { todo: mockTodo },
    })

    expect(wrapper.text()).toContain('写周报')
  })

  it('双击应该触发 activate 事件', async () => {
    const wrapper = mount(TodoTag, {
      props: { todo: mockTodo },
    })

    await wrapper.trigger('dblclick')
    expect(wrapper.emitted('activate')).toBeTruthy()
  })

  it('应该根据状态显示不同样式', async () => {
    const wrapper = mount(TodoTag, {
      props: { todo: mockTodo, isActive: true },
    })

    expect(wrapper.classes()).toContain('active')
  })
})
```

## 6. 测试执行时机

| 时机 | 测试类型 | 说明 |
|------|----------|------|
| 开发过程中 | 单元测试 | 每完成一个功能就运行 |
| 提交代码前 | 单元测试 + 集成测试 | 确保不破坏现有功能 |
| 合并到 main | 所有测试 | 完整回归测试 |
| 发布前 | 手动测试清单 | 全面验证 |

## 7. 测试覆盖率目标

| 层级 | 目标覆盖率 | 说明 |
|------|------------|------|
| 工具函数 | >= 90% | 核心逻辑必须覆盖 |
| 组合式函数 | >= 80% | 关键业务逻辑覆盖 |
| Vue 组件 | >= 60% | 覆盖主要交互场景 |
| Rust 命令 | >= 70% | 覆盖主要命令 |
| **整体** | **>= 70%** | 个人项目合理目标 |
