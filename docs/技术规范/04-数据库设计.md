# 数据库设计规范

> 本文档定义 YozaPomo 使用 SQLite 数据库的表结构、索引和迁移策略。

---

## 1. 数据库概述

- **引擎**：SQLite 3
- **访问方式**：`tauri-plugin-sql`
- **存储位置**：应用数据目录（`tauri::api::path::app_data_dir`）
- **数据库文件**：`yozapomo.db`
- **编码**：UTF-8

## 2. 表结构定义

### 2.1 `todos` — TODO 项

```sql
CREATE TABLE todos (
    id            TEXT PRIMARY KEY,           -- UUID
    title         TEXT NOT NULL,              -- 标题（必填）
    date          TEXT NOT NULL,              -- 日期 YYYY-MM-DD
    notes         TEXT DEFAULT '',            -- 备注（Markdown 格式）
    completed     INTEGER DEFAULT 0,          -- 是否完成 0/1
    sort_order    INTEGER DEFAULT 0,          -- 排序序号
    created_at    TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
    updated_at    TEXT NOT NULL DEFAULT (datetime('now', 'localtime'))
);

CREATE INDEX idx_todos_date ON todos(date);
CREATE INDEX idx_todos_completed ON todos(completed);
```

### 2.2 `pomodoro_sessions` — 专注会话

```sql
CREATE TABLE pomodoro_sessions (
    id              TEXT PRIMARY KEY,         -- UUID
    todo_id         TEXT,                     -- 关联的 TODO（可为空 = 独立专注）
    start_time      TEXT NOT NULL,            -- 开始时间 ISO 8601
    end_time        TEXT,                     -- 结束时间 ISO 8601
    planned_duration INTEGER NOT NULL,        -- 计划时长（秒）
    actual_duration  INTEGER DEFAULT 0,       -- 实际专注时长（秒）
    status          TEXT DEFAULT 'focusing',  -- focusing/paused/completed/abnormal
    notes           TEXT DEFAULT '',          -- 专注笔记（Markdown）
    created_at      TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
    FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE SET NULL
);

CREATE INDEX idx_sessions_todo ON pomodoro_sessions(todo_id);
CREATE INDEX idx_sessions_start ON pomodoro_sessions(start_time);
CREATE INDEX idx_sessions_status ON pomodoro_sessions(status);
```

### 2.3 `settings` — 应用设置

```sql
CREATE TABLE settings (
    key     TEXT PRIMARY KEY,                 -- 设置键名
    value   TEXT NOT NULL,                    -- 设置值（JSON 字符串）
    updated_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime'))
);
```

**默认设置值**：

| key | 默认值 | 说明 |
|-----|--------|------|
| `focus_duration` | `1500` | 默认专注时长（秒），25分钟 |
| `todo_display_count` | `3` | 悬浮窗显示 TODO 数量 |
| `show_countdown` | `true` | 是否显示倒计时 |
| `notification_sound` | `default` | 通知音效文件路径 |
| `rest_duration` | `300` | 默认休息时长（秒），5分钟 |

## 4. 查询规范

### 4.1 常用查询模板

```typescript
// 获取今日 TODO
const todayTodos = await db.select<Todo[]>(
  'SELECT * FROM todos WHERE date = ? ORDER BY sort_order ASC',
  [today]
)

// 获取未完成的昨日 TODO（用于"昨日未完成"页面）
const yesterdayIncomplete = await db.select<Todo[]>(
  'SELECT * FROM todos WHERE date = ? AND completed = 0 ORDER BY sort_order ASC',
  [yesterday]
)

// 获取 TODO 的专注历史
const sessions = await db.select<FocusSession[]>(
  'SELECT * FROM pomodoro_sessions WHERE todo_id = ? ORDER BY start_time DESC',
  [todoId]
)

// 获取今日专注统计
const todayStats = await db.select<{ total_minutes: number; session_count: number }[]>(
  `SELECT
    SUM(actual_duration) / 60 as total_minutes,
    COUNT(*) as session_count
  FROM pomodoro_sessions
  WHERE date(start_time) = date('now', 'localtime')
    AND status = 'completed'`
)
```

### 4.2 事务规范

```typescript
// ✅ 正确：使用事务处理多表操作
async function endTodoWithSession(todoId: string, session: FocusSession) {
  await db.execute('BEGIN TRANSACTION')
  try {
    await db.execute(
      'UPDATE todos SET completed = 1, updated_at = datetime("now", "localtime") WHERE id = ?',
      [todoId]
    )
    await db.execute(
      'UPDATE pomodoro_sessions SET end_time = datetime("now", "localtime"), actual_duration = ?, status = "completed" WHERE id = ?',
      [session.actualDuration, session.id]
    )
    await db.execute('COMMIT')
  } catch (error) {
    await db.execute('ROLLBACK')
    throw error
  }
}
```

## 5. 数据迁移策略

### 5.1 版本管理

```sql
-- 迁移版本表
CREATE TABLE IF NOT EXISTS migrations (
    version INTEGER PRIMARY KEY,
    applied_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime'))
);
```

### 5.2 迁移命名

```
migrations/
├── 001_initial_schema.sql      -- 初始建表
├── 002_add_tags_column.sql     -- 后续迁移
└── ...
```

### 5.3 迁移执行

```rust
// src-tauri/src/db/migrations.rs
pub fn run_migrations(conn: &Connection) -> Result<(), AppError> {
    conn.execute_batch("CREATE TABLE IF NOT EXISTS migrations (
        version INTEGER PRIMARY KEY,
        applied_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime'))
    )")?;

    let current_version: i32 = conn
        .query_row("SELECT COALESCE(MAX(version), 0) FROM migrations", [], |row| row.get(0))?;

    // 按版本号顺序执行迁移
    if current_version < 1 {
        conn.execute_batch(include_str!("../../migrations/001_initial_schema.sql"))?;
        conn.execute("INSERT INTO migrations (version) VALUES (1)", [])?;
    }
    // 后续迁移...

    Ok(())
}
```

## 6. 数据安全

1. **UUID 生成**：使用 `uuid` crate 生成 v4 UUID，确保全局唯一
2. **SQL 注入防护**：所有查询使用参数化查询，禁止字符串拼接
3. **备份策略**：每次发版前手动备份数据库文件
4. **数据清理**：超过 90 天的已完成 TODO 可自动归档（可选功能）
