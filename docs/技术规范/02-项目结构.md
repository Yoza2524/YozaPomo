# 项目结构规范

> 本文档定义 YozaPomo 的目录结构、文件命名和模块划分标准。

---

## 1. 顶层目录结构

```
YozaPomo/
├── docs/                          # 项目文档（已有）
│   ├── 技术规范/                   # 技术标准文档
│   ├── 设计规范/                   # UI/交互设计文档
│   ├── 开发流程/                   # 流程与规范文档
│   ├── 需求文档/                   # PRD 需求文档
│   ├── 开发日志/                   # 每日开发记录
│   └── 乱七八糟/                   # 杂项笔记
├── src/                           # 前端源码（Vue 3）
├── src-tauri/                     # Rust 后端源码
├── public/                        # 静态资源
├── CLAUDE.md                      # 项目指引中枢
├── README.md                      # 项目说明
├── package.json                   # 前端依赖
├── pnpm-lock.yaml                 # 锁文件
├── tsconfig.json                  # TypeScript 配置
├── vite.config.ts                 # Vite 构建配置
├── tailwind.config.js             # Tailwind 配置
├── .eslintrc.cjs                  # ESLint 配置
├── .prettierrc                    # Prettier 配置
├── .gitignore                     # Git 忽略规则
└── Cargo.toml                     # Rust 依赖（src-tauri 内）
```

## 4. 文件命名规范

### 前端文件
| 类型 | 命名格式 | 示例 |
|------|----------|------|
| Vue 组件 | PascalCase `.vue` | `FloatingWindow.vue` |
| 组合式函数 | camelCase `use*.ts` | `useTimer.ts` |
| Pinia Store | camelCase `*Store.ts` | `todoStore.ts` |
| 类型定义 | camelCase `.ts` | `todo.ts` |
| 工具函数 | camelCase `.ts` | `format.ts` |
| 样式文件 | kebab-case `.css` | `floating-window.css` |

### Rust 文件
| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 模块文件 | snake_case `.rs` | `todo_commands.rs` |
| 命令函数 | snake_case | `create_todo` |
| 数据库表 | snake_case 复数 | `todos`, `pomodoro_sessions` |

## 5. 模块划分原则

1. **按功能域划分**：`floating/`、`management/` 对应两个主要界面
2. **组合式函数封装可复用逻辑**：每个 composable 职责单一
3. **类型定义独立**：所有 TypeScript 类型集中在 `types/` 目录
4. **Rust 模块对应前端功能**：`commands/` 中的每个文件对应一个功能域
5. **禁止循环依赖**：组件 → composables → stores → utils，单向依赖
