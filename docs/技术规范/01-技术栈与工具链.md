# 技术栈与工具链规范

> 本文档定义 YozaPomo 项目的技术选型、版本要求和工具链配置标准。

---

## 1. 核心技术栈

| 层级 | 技术 | 版本要求 | 说明 |
|------|------|----------|------|
| 桌面框架 | Tauri v2 | `>= 2.0` | 跨平台桌面应用框架（Rust） |
| 前端框架 | Vue 3 | `>= 3.4` | 使用 Composition API + `<script setup>` |
| 类型系统 | TypeScript | `>= 5.4` | 严格模式（`strict: true`） |
| UI 组件库 | Naive UI | `>= 2.39` | 主力 UI 组件 |
| 样式方案 | Tailwind CSS | `>= 3.4` | 原子化 CSS，配合 Naive UI 使用 |
| 状态管理 | Pinia | `>= 2.1` | 替代 Vuex，TypeScript 友好 |
| 数据库 | SQLite | — | 通过 `tauri-plugin-sql` 访问 |
| 音频播放 | Howler.js | `>= 2.2` | 跨平台音频播放，支持 mp3/wav |
| 动画系统 | Canvas + CSS | — | 粒子动画用 Canvas，UI 动画用 CSS |

## 2. 开发工具链

| 工具 | 用途 | 备注 |
|------|------|------|
| VS Code | IDE | 主力开发环境 |
| Claude Code | AI 辅助编程 | 通过 CC-Switch 路由到 MIMO 2.5 |
| Git | 版本控制 | `>= 2.40` |
| pnpm | 包管理器 | 优先于 npm/yarn，节省磁盘空间 |
| Vite | 构建工具 | Tauri v2 默认集成 |
| ESLint | 代码检查 | 配合 `@typescript-eslint` |
| Prettier | 代码格式化 | 统一代码风格 |
| rustfmt | Rust 格式化 | Rust 代码格式化工具 |
| clippy | Rust Lint | Rust 代码质量检查 |

## 3. Tauri v2 插件清单

| 插件 | crate 名 | 功能 |
|------|-----------|------|
| SQL | `tauri-plugin-sql` | SQLite 数据库访问 |
| Shell | `tauri-plugin-shell` | 系统托盘交互 |
| Store | `tauri-plugin-store` | 本地键值存储（设置） |
| Dialog | `tauri-plugin-dialog` | 文件选择对话框 |
| Notification | `tauri-plugin-notification` | 系统通知（备选方案） |

## 4. 版本锁定原则

- `package.json` 中所有依赖使用精确版本或 `^` 范围，禁止使用 `~` 或 `*`
- `Cargo.toml` 中依赖使用精确版本
- 升级任何核心依赖前，必须在开发日志中记录升级原因和影响评估
- 每次发版前执行完整回归测试

## 5. 环境要求

```
Node.js >= 20 LTS
Rust >= 1.75 (stable channel)
pnpm >= 9.0
```
