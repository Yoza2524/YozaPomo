## 🔴 Critical（必须修复）

### C1. 连续删除时旧兜底定时器污染第二次动画

**文件**：`src/components/floating/TodoList.vue:58-108`
**维度**：正确性

**描述**：当 `shatterTodoId` 在上一次删除动画仍在运行时发生变化，watcher 会直接覆盖 `snapshotTodos`、`collapsingTodoId`、`appearingTodoId`，但不会先调用 `resetAnimState()`。旧的 fallback timer（line 104 设置的 500ms 定时器）会存活。当它触发时，会调用 `finalizeAnimation()` 重置正在进行的第二次动画状态，对错误的 todo 发出 `shatterDone` 事件。

**建议**：在 `shatterTodoId` watcher 开头（line 59），在检查 `todoInList` 之前，先调用 `resetAnimState()` 清除任何正在进行的动画和 fallback timer。

---

### C2. useAutoResize 事件监听器从未移除（内存泄漏）

**文件**：`src/composables/useAutoResize.ts:119-147`
**维度**：正确性 / 性能

**描述**：`onMounted` 中使用匿名箭头函数添加了 5 个事件监听器（`animationstart`、`animationend`、`transitionstart`、`transitionend`、`transitioncancel`），但 `onUnmounted`（line 150-161）只清理了 ResizeObserver、debounce timer 和 rAF loop，从未移除这 5 个监听器。由于是匿名函数，`removeEventListener` 无法定位它们。

**建议**：使用 `AbortController` 批量移除：
```ts
const ac = new AbortController()
el.addEventListener('animationstart', handler, { signal: ac.signal })
// ...
onUnmounted(() => ac.abort())
```

---

### C3. IPC 调用在每帧动画时触发（~120 次/秒）

**文件**：`src/composables/useAutoResize.ts:24-41`
**维度**：性能

**描述**：`resizeImmediate()` 在每个 rAF 帧调用 `getCurrentWindow()` 然后 `Promise.all([win.outerSize(), win.outerPosition()])`。60fps 下每秒产生 120 次 Tauri IPC 桥接调用，每次跨越 JS-Rust 边界。此外 `Promise.all` 异步解析，多个重叠的 promise chain 可能同时在飞，`.then()` 回调乱序会导致抖动。

**建议**：
1. 在模块作用域缓存 `getCurrentWindow()`（FloatingWindow.vue line 21 已有此模式）
2. 只在 `getBoundingClientRect()` 返回值实际变化时才调用 IPC
3. 考虑缓存 size/position，仅在内容尺寸变化时重新获取

## 🟠 Major（强烈建议修复）

### M1. FloatingWindow.vue onMoved 监听器从未清理

**文件**：`src/components/floating/FloatingWindow.vue:97-102`
**维度**：正确性

**描述**：`appWindow.onMoved()` 返回 `Promise<UnlistenFn>` 但返回值被丢弃。该监听器在 `onUnmounted` 中从未被移除，与其他 4 个监听器（line 105-110）的清理模式不一致。

**建议**：存储 unlisten 函数并在 `onUnmounted` 中调用。

---

### M2. startRest() 在 completeFocus() 失败时仍被调用

**文件**：`src/components/floating/FocusButton.vue:37-39`
**维度**：正确性

**描述**：超时分支中 `await focusStore.completeFocus()` 后直接调用 `focusStore.startRest()`。如果 `completeFocus()` 返回 false（数据库写入失败），`startRest()` 仍会被调用，UI 进入 'resting' 状态但数据库中的 focus session 仍不完整，状态不一致。

**建议**：检查返回值：`const success = await focusStore.completeFocus(); if (success) focusStore.startRest();`

---

### M3. 重复的 resize 逻辑：resizeImmediate() vs updateWindowSize()

**文件**：`src/composables/useAutoResize.ts:24-100`
**维度**：代码质量

**描述**：两个函数共享几乎相同的逻辑：读取容器尺寸、计算 contentWidth/contentHeight、获取窗口 outerSize/outerPosition、调用 setSize/setPosition。唯一区别是 `updateWindowSize` 有屏幕边界 clamp 和错误日志。

**建议**：提取共享的测量 + 调整逻辑为单一内部函数，`resizeImmediate()` 调用时不 clamp（动画帧性能优先），`updateWindowSize()` 调用时加 clamp + 错误处理。

---

### M4. TodoList 动画状态机应抽取为 composable

**文件**：`src/components/floating/TodoList.vue:43-164`
**维度**：代码质量

**描述**：删除动画状态机使用 8 个 ref 变量 + 6 个函数，约 120 行编排逻辑嵌入组件中，难以测试和理解。

**建议**：抽取为 `useDeleteAnimation(todos, maxDisplay)` composable，返回 `{ collapsingTodoId, appearingTodoId, isLastTagCollapsing, ellipsisFadingOut, snapshotTodos, triggerShatter, handleCollapseDone, handleAppearDone }`。

### M5. TodoTag 粒子物理逻辑应抽取为 composable

**文件**：`src/components/floating/TodoTag.vue:8-18, 100-162`
**维度**：代码质量

**描述**：`Particle` 接口和 `startShatter()` 函数包含自包含的粒子物理模拟（重力、扩散、速度、旋转、渐隐），约 60 行可复用的物理代码嵌入组件中。

**建议**：抽取为 `useParticleBurst(config)` composable，返回 `{ particles, start, isRunning }`。

### M6. 18 个 DOM 粒子逐帧样式变更，应改用 Canvas

**文件**：`src/components/floating/TodoTag.vue:215-231`
**维度**：性能

**描述**：粉碎动画通过 Teleport 渲染 18 个独立 DOM 元素，每个元素有 7 个内联样式属性在 rAF 循环中每帧变更。每秒约 1080 次样式变更，每次触发浏览器 layout/paint。`ParticleCanvas.vue` 已有 Canvas 实现。

**建议**：用类似 `ParticleCanvas.vue` 的 Canvas 方案替代。单个 `<canvas>` 元素用 `ctx.arc()` 绘制，避免所有 DOM 变更开销。

### M7. 过度宽泛的权限：shell:default

**文件**：`src-tauri/capabilities/default.json:25`
**维度**：安全性

**描述**：capability 文件包含 `shell:default`，授予广泛的 shell 访问权限。Rust 后端和前端代码中没有使用 `Command::new()` 或 shell 执行的证据。前端使用 `@tauri-apps/plugin-opener` 而非 `@tauri-apps/plugin-shell`。不必要的 shell 权限增加了攻击面。

**建议**：从 `default.json` 移除 `shell:default`，从 `lib.rs` 移除 `.plugin(tauri_plugin_shell::init())`。如需保留，改用更具体的权限如 `shell:allow-open`。

### M8. todo-deleted 事件未做类型校验

**文件**：`src/components/floating/FloatingWindow.vue:78-79`
**维度**：安全性 / 正确性

**描述**：`todo-deleted` 事件处理器直接将 `event.payload` 赋值给 `shatterTodoId.value`，未检查 payload 是否为 string。对比 `todo-changed` 处理器（line 68-74）正确使用了 `typeof event.payload === 'string'` 检查。

### M9. Markdown 渲染存在存储型 XSS 风险

**文件**：`src/components/management/TodoDetail.vue:39-46, 88, 130`
**维度**：安全性

**描述**：`renderMarkdown()` 将用户输入的 `todo.notes` 和 `session.notes` 直接通过 `marked.parse()` 处理，未做 HTML 消毒。输出通过 `v-html` 注入 DOM。`marked` 库默认不剥离原始 HTML。恶意 notes 如 `<img src=x onerror=alert(document.cookie)>` 会在管理窗口中执行。

**建议**：安装并使用 `DOMPurify` 消毒 markdown 输出：`return DOMPurify.sanitize(marked.parse(text) as string)`。

## 🟡 Minor（建议修复）

### m1. transitionend 在兜底定时器之后触发的非确定性竞态

**文件**：`src/components/floating/TodoList.vue:103-108`
**描述**：CSS transition 0.4s 但 fallback timer 500ms，margin 仅 100ms。慢机器或后台标签页节流时，transitionend 可能在 500ms 后才触发。现有 guard 保证安全，但余量不足。
**建议**：将 fallback timer 增加到 600-800ms。

### m2. animCount 可能因事件冒泡导致双重计数

**文件**：`src/composables/useAutoResize.ts:118-147`
**描述**：子元素的 animationstart 事件冒泡到容器，可导致 animCount 被多次递增。若元素在动画期间从 DOM 移除，animationend 永不触发，animCount 永久 > 0，rAF 无限循环。
**建议**：用 Set 跟踪 animationName 而非简单计数器。

### m3. focusStore.isComputed 是死代码

**文件**：`src/stores/focusStore.ts:27`
**描述**：`isCompleted` 检查 `status === 'completed'`，但 store 从不将 status 设为 'completed'（`completeFocus` 设为 'idle'），该 getter 永远返回 false 且无组件引用。
**建议**：移除 `isCompleted` computed 属性。

### m4. CSS 动画时长在多处硬编码

**文件**：`src/components/floating/TodoTag.vue:289,300,328,352` + `TodoList.vue:343,345,415`
**描述**：0.4s、0.6s 和 `cubic-bezier(0.4, 0, 0.2, 1)` 在 2 个文件 6+ 处硬编码。JS fallback timer 也使用 500ms。
**建议**：定义 CSS 自定义属性 `--anim-duration-normal: 0.4s` 等，JS 端从共享常量派生。

### m5. FloatingWindow 传递未声明的 disabled prop

**文件**：`src/components/floating/FloatingWindow.vue:308`
**描述**：传递 `:disabled="false"` 给 FocusButton，但 FocusButton 未声明 `disabled` prop。
**建议**：移除 `:disabled="false"` 或在 FocusButton 中添加 prop 声明。

### m6. 动态导入 emit 冗余

**文件**：`src/components/floating/FloatingWindow.vue:93, 203`
**描述**：两处使用 `const { emit } = await import('@tauri-apps/api/event')` 动态导入，而同模块的 `emitTo` 已在 line 13 静态导入。
**建议**：将 `emit` 添加到静态导入，移除两处动态导入。

### m7. todoStore 中不安全的类型断言

**文件**：`src/stores/todoStore.ts:94`
**描述**：`UpdateTodoInput` 被 cast 为 `Record<string, unknown>` 传给 `db.updateTodo`，绕过了 TypeScript 类型检查。
**建议**：让 `db.updateTodo` 直接接受 `UpdateTodoInput` 类型。

### m8. 无输入长度验证

**文件**：`src-tauri/src/db/schema.rs:18-25, 28-36`
**描述**：`CreateTodoInput` 和 `UpdateTodoInput` 的 `title` 和 `notes` 字段无长度限制。前端 `NInput` 也无 `maxlength` 属性。
**建议**：Rust 端添加验证（title ≤ 500 字符，notes ≤ 10000 字符），前端添加 `maxlength`。

### m9. set_setting 无 key 白名单

**文件**：`src-tauri/src/commands/settings_commands.rs:42-56`
**描述**：`set_setting` 接受任意 `key` 写入数据库，无服务端校验。
**建议**：Rust 端添加有效 key 白名单，拒绝未知 key。

### m10. 粒子物理使用固定 dt 而非实际帧时间

**文件**：`src/components/floating/TodoTag.vue:141`
**描述**：`const dt = 1 / 60` 硬编码。144Hz 显示器上粒子以 2.4 倍速移动，慢机器上粒子过慢。
**建议**：用 `performance.now()` 计算实际 dt。

### m11. watch(hiddenCount) 中的 setTimeout 未清理

**文件**：`src/components/floating/TodoList.vue:187-189`
**描述**：350ms setTimeout 清除 `ellipsisAnimClass`，未在 `onUnmounted` 中清理。
**建议**：跟踪 timer 并在 `onUnmounted` 中清除。

### m12. 模板中直接调用 getBoundingClientRect() 4 次

**文件**：`src/components/floating/TodoList.vue:296-297`
**描述**：省略号粒子的 Teleport 中 `:style` 绑定调用 `ellipsisRef.getBoundingClientRect()` 4 次，强制浏览器计算布局。
**建议**：用 computed 一次性计算 `{ left, top }`。

## 💡 Suggestion（可选优化）

### S1. finalizeAnimation 优先使用 appearingTodoId 语义不正确

**文件**：`src/components/floating/TodoList.vue:143`
**描述**：`appearingTodoId || collapsingTodoId` 中，appearing 是出现的隐藏 todo，不是被删除的。虽然父组件只用 ID 触发 fetch 所以不影响功能，但语义错误。
**建议**：改为 `collapsingTodoId || appearingTodoId` 或传递原始 `shatterTodoId`。

### S2. deep watcher 导致不必要的深层遍历

**文件**：`src/components/floating/TodoList.vue:207-212`
**描述**：`{ deep: true }` 在整个 todos 数组变更时深层遍历，但逻辑只检查 `newTodoId` 和长度变化。
**建议**：移除 `deep: true`，分别 watch `props.newTodoId` 和 `props.todos.length`。

### S3. FocusButton 重复的 CSS 规则

**文件**：`src/components/floating/FocusButton.vue:120-122, 173-175`
**描述**：`.action-button.active` 定义了两次（`calc(50% - 4px)` 和 `50%`），后者覆盖前者。
**建议**：移除 line 120-122 的死代码。

### S4. 每个 TodoTag 实例独立调用 getCurrentWindow()

**文件**：`src/components/floating/TodoTag.vue:58`
**描述**：N 个可见 tag 创建 N 个 Window wrapper 对象。Window 对象轻量但可优化。
**建议**：通过 `provide/inject` 从 FloatingWindow 共享实例。

### S5. snapshotTodos 每次删除创建完整数组拷贝

**文件**：`src/components/floating/TodoList.vue:70`
**描述**：`[...props.todos]` 浅拷贝触发 5 个 computed 重新计算。小列表可接受。
**建议**：当前无需优化，若列表增长可考虑直接从 snapshot 计算。
