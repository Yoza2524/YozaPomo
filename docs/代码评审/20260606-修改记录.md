## C1. 连续删除时旧兜底定时器污染第二次动画 ✅

**文件**：`src/components/floating/TodoList.vue`
**修复**：在 `shatterTodoId` watcher 开头添加 `resetAnimState()` 调用，确保每次新删除都从干净状态开始。

### 修改内容

在 `watch(() => props.shatterTodoId, ...)` 回调的 `if (!id) return` 之后、业务逻辑之前，插入一行 `resetAnimState()`，清除上一次动画可能残留的状态（包括 fallback timer、collapsingTodoId、appearingTodoId、snapshotTodos 等）。

### 影响范围

- **直接**：`TodoList.vue` 的删除动画状态机
- **间接**：`FloatingWindow.vue` 的 `handleShatterDone` 回调（由 `shatterDone` 事件触发）
- **不受影响**：新建动画、省略号动画、专注状态、右键菜单

### 需要测试的内容

1. **连续删除可见 TODO**（核心场景）
   - 创建 3 个 TODO → 删除第 3 个 → 动画完成后删除第 2 个 → 动画完成后删除第 1 个
   - 每次删除应独立播放完整动画（粒子 → 收起 → 上移/省略号出现）
   - 最后一个删除后应显示"暂无待办"

2. **快速连续删除**（边界场景）
   - 创建 3 个 TODO → 快速连续删除（上一个动画未完成时触发下一个）
   - 动画可能重叠，但不应出现：悬浮窗冻结、显示残留 TODO、窗口高度异常

3. **删除隐藏项后删除可见项**
   - 创建 4+ 个 TODO（maxDisplay=3）→ 删除第 4 个（隐藏项，省略号粒子）→ 删除第 1 个（可见项）
   - 省略号粒子播放后，可见项的 collapse + appear 动画应正常

4. **删除后新建**
   - 删除一个 TODO → 动画完成后新建一个 TODO
   - 新建动画应正常播放，不被残留状态影响

## C3. rAF 每帧 IPC 调用优化 ⚠️ 部分回退

**文件**：`src/composables/useAutoResize.ts`
**状态**：`animCount` 机制仍在（动画期间 ResizeObserver 跳过 + rAF 驱动），但 `getCurrentWindow()` 缓存和 `lastContentHeight/lastContentWidth` 尺寸缓存已被移除。当前 `resizeImmediate()` 每次调用 `getCurrentWindow()` 并执行 `outerSize()` / `outerPosition()` IPC。由于 `animCount` 机制确保只在动画期间才高频调用，影响有限，但仍有优化空间。

### 当前实际行为

- 动画期间：rAF 每帧调用 `resizeImmediate()`，每次都执行 3 次 IPC（getCurrentWindow + outerSize + outerPosition）
- 非动画期间：ResizeObserver + debounce 16ms，调用频率低，影响小

### 潜在优化（未实施）

1. `getCurrentWindow()` 可提升到模块作用域，避免重复创建引用
2. 可缓存 `lastContentHeight/lastContentWidth`，尺寸未变时跳过 IPC
